Por que Robótica na TI?

Roteiro

O vídeo apresentado contém uma breve explicação sobre a origem da palavra "robô", originada da língua tcheca, falamos sobre os materiais que compõem um robô, como eles são utilizados nas áreas industriais, domésticas etc. As vantagens e as desvantagens da utilização de robôs também foi um tópico debatido, em questão de custo benefício, mão de obra, programação do robô e outros. Apresentamos como a TI atua na programação do software presentes nos robôs, e os tipos de software que são utilizados nos mesmos, finalizando o vídeo com uma curta previsão sobre o desenvolvimento da robótica no futuro. Para a criação do vídeo, os integrantes gravaram os áudios de seus respectivos tópicos, foi então utilizado para a edição do vídeo o programa: Vegas Pro 18.0. Todos os vídeos e imagens utilizados de fundo foram retirados da internet com licenciamento grátis para uso, o site Powtoon foi utilizado para pequenos trechos do vídeo que envolviam animação.

Componentes usados em aula

Utilizado no desenvolvimento de protótipos, permitindo o desenvolvimento de projetos autônomos.

Utilizada para fazer as conexões do circuito.

Utilizada para alimentar o circuito.

Responsável pela movimentação do carrinho.

LDR

Responsável pela movimentação dos motores através da luminosidade.

Resistor

Responsável pela limitação da corrente elétrica, evitando picos de energia.

Led

Utilizado para sinalização.

Ponte H

Responsável pelo controle dos motores.

Seguidor de linha

Aqui é encontrado a estrutura do robô seguidor de linha, o robô segue o princípio de usar a luz como condutor, quando o sensor capta a linha branca, o robô entende a parte branca da linha como "1" se o robô não encontra a linha, o valor "0" é retornado, indicando que o sensor não esta na linha. O algoritimo PID foi implementado, que tem como objetivo de controlar o desempenho do robô para melhor a sua eficiência.

void controleMotor() {    // Criamos uma função para controle do driver de motor

int velocidade_direita;



   // Definimos as portas digitais

   // Ajustes motor da esquerda

digitalWrite (PININ3, HIGH);
digitalWrite (PININ4, LOW);
} else {
digitalWrite (PININ3, LOW);

}

   // Ajustes motor da direita

velocidade_direita = -velocidade_direita;
digitalWrite (PININ1, LOW);
digitalWrite (PININ2, HIGH);
} else {
digitalWrite (PININ1, HIGH);
digitalWrite (PININ2, LOW);
}

analogWrite (PINENA, velocidade_esquerda);
analogWrite (PINENB, velocidade_direita);
}

void opcaoMotor() {     //Função para controlar o motor com pre definições para testes.

char option

case '4':   // Caso 4 seja selecionado, o robô vira vira para a direita.

case '2':   // Caso 2 seja selecionado, o robô vai para trás.

break;
case '0':  // Caso 0 seja selecionado, o robô para.

break;
}
}

  //O PID é calculado conforme o erro.
void calcula_pid()
{
P = error;
I = I + previous_I;
D = error - previous_error;

valor_PID = (Kp * P) + (Ki * I) + (Kd * D);

previous_I = I;
previous_error = error;
}

//Controla os motores conforme valor do PID
void controla_motor_pid()
{
   // Calcula a eficiência da velocidade do motor
int velocidade_motor_esquerdo = velocidade_motor_inical - valor_PID;
int velocidade_motor_direito = velocidade_motor_inical + valor_PID;

   // Usada a função "constrain" para restringir a velocidade dos motores entre 0 e 255
constrain(velocidade_motor_esquerdo, 0, 255);
constrain(velocidade_motor_direito, 0, 255);

Serial.print("Motor esquerda: ");
Serial.print(velocidade_motor_esquerdo);
Serial.print(" ");
Serial.print("Motor direita: ");
Serial.println(velocidade_motor_direito);

controleMotor(velocidade_motor_esquerdo, velocidade_motor_direito);
}
